# 电商购物平台 — 技术说明文档

---

## （1）登录验证原理说明

### 整体架构

本系统采用 **JWT（JSON Web Token）+ Apache Shiro** 实现无状态的身份认证与授权。

```
用户浏览器                 后端服务器
    |                         |
    |  POST /api/auth/login   |
    |  {username, password}   |
    |------------------------>|
    |                         | 1. 查询数据库验证用户名密码（BCrypt）
    |                         | 2. 生成 JWT Token（含 userId, username, role）
    |  {token, userId, role}  |
    |<------------------------|
    |                         |
    |  后续请求                |
    |  Authorization: Bearer  |
    |  <token>                |
    |------------------------>|
    |                         | 3. JwtFilter 拦截，提取 Bearer Token
    |                         | 4. JwtRealm 解析 JWT，验证签名和有效期
    |                         | 5. 构造 UserPrincipal 存入 Subject
    |                         | 6. UserContext 从 Subject 获取用户信息
    |                         | 7. Service 层判断角色权限
    |  Result<T> JSON 响应     |
    |<------------------------|
```

### 详细流程

**Step 1: 登录请求**

用户通过 `POST /api/auth/login` 提交用户名和密码。该接口在 Shiro 配置中设为 `anon`（无需认证）。

`AuthServiceImpl.login()` 执行：
1. 用 `UserMapper.findByUsername()` 查询数据库
2. 用 `BCryptPasswordEncoder.matches()` 比对密码（数据库中存储的是 BCrypt 密文）
3. 验证通过后，调用 `JwtUtils.createToken(userId, username, role)` 生成 JWT

**Step 2: JWT 令牌生成**

`JwtUtils` 使用 `io.jsonwebtoken` 库生成 Token：
```
Header: { "alg": "HS256", "typ": "JWT" }
Payload: {
    "userId": 1,
    "username": "admin",
    "role": "admin",
    "sub": "admin",
    "exp": 当前时间 + 72小时
}
Signature: HMAC-SHA256(base64(Header) + "." + base64(Payload), secret密钥)
```

Token 通过 HS256 算法签名，密钥配置在 `application.yml` 的 `jwt.secret` 中。

**Step 3: 前端存储 Token**

前端 `user.js` (Pinia Store) 将 token、userId、username、role 存入 `localStorage`，实现页面刷新后仍保持登录状态。每次请求通过 Axios 拦截器自动在 Header 添加 `Authorization: Bearer <token>`。

**Step 4: 请求拦截与验证**

每个需要认证的请求到达后端时：

1. **JwtFilter（Shiro 过滤器）** 从 `Authorization` 请求头提取 `Bearer ` 后的 Token
2. 构造 `JwtToken` 对象，调用 `Subject.login(jwtToken)` 委托给 Shiro
3. **JwtRealm.doGetAuthenticationInfo()** 执行真正的认证：
   - 调用 `JwtUtils.parse(token)` 解析 JWT
   - 验证签名（防止篡改）
   - 验证过期时间（`exp` 字段）
   - 提取 `userId`、`username`、`role`
   - 构造 `UserPrincipal` 对象作为认证主体
4. 认证成功后，`UserContext` 工具类可从 `SecurityUtils.getSubject().getPrincipal()` 获取 `UserPrincipal`，进而读取用户 ID 和角色

**Step 5: 权限校验**

- `UserContext.isAdmin()` 判断角色是否为 `admin`
- 各 `ServiceImpl` 在敏感操作前调用 `requireAdmin()` 检查权限
- Shiro 过滤链中 `/api/**` 路径需要 jwt 认证，未认证返回 401

### 安全特性

| 特性 | 实现方式 |
|------|----------|
| 密码加密 | BCrypt（`spring-security-crypto`） |
| 防篡改 | JWT HMAC-SHA256 签名 |
| 过期控制 | JWT `exp` 字段，默认 72 小时 |
| 无状态 | 禁用 Shiro Session，每次请求重新验证 JWT |
| 跨域 | `CorsInterceptor` 添加 CORS 响应头 |

### 关键代码文件

| 文件 | 职责 |
|------|------|
| `config/ShiroConfig.java` | Shiro 过滤器链配置，禁用 Session |
| `shiro/JwtFilter.java` | 提取 Bearer Token，触发认证 |
| `shiro/JwtRealm.java` | 解析 JWT，构造 UserPrincipal |
| `shiro/UserPrincipal.java` | 用户主体信息（userId/username/role） |
| `util/JwtUtils.java` | JWT 生成和解析 |
| `util/UserContext.java` | 从 Subject 获取当前用户 |
| `service/impl/AuthServiceImpl.java` | 登录/注册业务逻辑 |

## （2）查询功能的前后端工作过程说明

以**商品列表查询**为例，说明前后端完整交互过程。

### 查询请求链路

```
前端 Vue 组件                Axios 拦截器              后端服务器
    |                           |                         |
    |  request.get(             |                         |
    |   '/api/products',        |                         |
    |   {params: {page,size,    |                         |
    |    name, category}})      |                         |
    |-------------------------->|                         |
    |                           | 1. 添加 Authorization   |
    |                           |    Header (如有token)   |
    |                           |------------------------>|
    |                           |                         | 2. 请求到达 DispatcherServlet
    |                           |                         | 3. RequestLoggingInterceptor
    |                           |                         |    打印 URL/方法/参数/IP
    |                           |                         | 4. Shiro 过滤链判断权限
    |                           |                         |    /api/products → anon (放行)
    |                           |                         | 5. ProductController.list()
    |                           |                         |    接收 @RequestParam 参数
    |                           |                         | 6. ProductService.query()
    |                           |                         |    boolean includeDeleted
    |                           |                         |      = adminView && isAdmin()
    |                           |                         | 7. ProductMapper.queryPage()
    |                           |                         |    SQL: SELECT ... WHERE
    |                           |                         |      name LIKE '%xxx%'
    |                           |                         |      AND category LIKE '%xxx%'
    |                           |                         |      AND price >= min
    |                           |                         |      AND price <= max
    |                           |                         |      AND deleted = 0
    |                           |                         |      LIMIT size OFFSET offset
    |                           |                         | 8. ProductMapper.countQuery()
    |                           |                         |      COUNT(1) 获取总数
    |                           |                         | 9. 封装 PageResult(total,page,
    |                           |                         |      size, records)
    |                           |                         | 10. Result.ok(pageResult)
    |                           |  {code:200,             |
    |                           |   message:"success",    |
    |                           |   data:{total,page,     |
    |                           |     size,records:[...]}}|
    |                           |<------------------------|
    |                           | 11. 检查 code === 200   |
    |                           | 12. 返回 body.data      |
    |<--------------------------|                         |
    | 13. 更新组件数据           |                         |
    |     list = res.records    |                         |
    |     total = res.total     |                         |
    | 14. 渲染表格和分页按钮    |                         |
```

### 数据流转详解

**① 前端发起请求**

`ProductList.vue` 中，用户点击搜索或切换分页时调用 `load()` 函数：

```javascript
const res = await request.get('/api/products', {
  params: { page: page.value, size: size.value, ...query.value }
})
list.value = res.data.records || []
total.value = res.data.total || 0
```

**② Axios 拦截器处理**

`request.js` 中配置了请求/响应拦截器：

- **请求拦截**：自动附加 `Authorization: Bearer <token>` 头
- **响应拦截**：
  - 检查 `res.data.code`，若不为 200 则 reject 为 Error
  - 返回 `res.data`（即后端 Result 对象）
  - 若收到 401 状态码，自动清除登录状态并跳转登录页

**③ 后端 Controller 层**

`ProductController.list()` 接收多条件查询参数：

```java
@GetMapping
public Result<PageResult<Product>> list(
    @RequestParam(required=false) String name,
    @RequestParam(required=false) String category,
    @RequestParam(required=false) BigDecimal minPrice,
    @RequestParam(required=false) BigDecimal maxPrice,
    @RequestParam(defaultValue="1") int page,
    @RequestParam(defaultValue="10") int size,
    @RequestParam(defaultValue="false") boolean adminView)
```

**④ Service 层业务处理**

`ProductServiceImpl.query()` 判断当前用户是否有权查看已下架商品，然后委托 Mapper 执行查询。

**⑤ Mapper 层 SQL 执行**

使用 MyBatis XML 动态 SQL，支持条件组合和分页：

```xml
<where>
    <if test="!includeDeleted">AND deleted = 0</if>
    <if test="name != null and name != ''">AND name LIKE '%' || #{name} || '%'</if>
    <if test="category != null and category != ''">AND category LIKE '%' || #{category} || '%'</if>
    <if test="minPrice != null">AND price >= #{minPrice}</if>
    <if test="maxPrice != null">AND price <= #{maxPrice}</if>
</where>
ORDER BY id DESC LIMIT #{limit} OFFSET #{offset}
```

**⑥ 响应返回链路**

`PageResult → Result.ok() → Controller → JSON序列化 → HTTP响应 → Axios拦截器 → Vue组件`

最终前端收到的数据结构：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "total": 50,
    "page": 1,
    "size": 10,
    "records": [
      { "id": 1, "name": "商品A", "price": 99.00, "stock": 100, ... },
      ...
    ]
  }
}
```

### 其他查询功能对比

| 功能 | 接口 | 认证 | 分页 | 说明 |
|------|------|------|------|------|
| 商品列表 | `GET /api/products` | 公开 | ✅ | 支持多条件模糊查询 |
| 商品详情 | `GET /api/products/{id}` | 公开 | - | 含评价列表 |
| 用户订单 | `GET /api/orders/my` | JWT | ✅ | 仅查看自己的订单 |
| 管理员订单 | `GET /api/orders/admin` | JWT+admin | ✅ | 查看所有订单 |
| 购物车 | `GET /api/cart` | JWT | - | 关联查询商品信息 |
| 评价列表 | `GET /api/reviews/product/{id}` | 公开 | - | 含用户名 |

### 统一返回格式

所有接口遵循 `Result<T>` 统一格式：

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | int | 200=成功，400=业务错误，401=未登录，403=无权限，500=服务器错误 |
| `message` | string | 提示信息 |
| `data` | T | 业务数据（可为 null） |

分页接口的 `data` 类型为 `PageResult<T>`：

| 字段 | 类型 | 说明 |
|------|------|------|
| `total` | long | 总记录数 |
| `page` | int | 当前页码 |
| `size` | int | 每页大小 |
| `records` | List<T> | 当前页记录列表 |
