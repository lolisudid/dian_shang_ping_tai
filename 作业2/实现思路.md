# 实现思路

## 1. 技术栈升级

将原项目从 **Spring Boot 2.2 + MyBatis + Shiro** 升级为 **Spring Boot 3.2.5 + MyBatis Plus 3.5.7 + JWT 拦截器**：

| 组件 | 旧版本 | 新版本 | 说明 |
|------|--------|--------|------|
| Spring Boot | 2.2.13 | 3.2.5 | Java 17+, Jakarta EE 9+ |
| ORM | MyBatis 3.5 | MyBatis Plus 3.5.7 | BaseMapper 减少样板代码 |
| 认证 | Shiro 1.7 | HandlerInterceptor | 更轻量，无状态 JWT |
| JWT | jjwt 0.9 | jjwt 0.12.6 | 新 API，支持密钥生成器 |
| 连接池 | Druid 1.2.8 | Druid 1.2.23 | Spring Boot 3 适配版 |
| Java | 1.8 | 17 | record、sealed、switch 表达式 |

## 2. 订单模块 CRUD 设计

### 核心思路

使用 MyBatis Plus 的 `BaseMapper<T>` 提供内置 CRUD 方法，复杂查询使用 `LambdaQueryWrapper` 构建类型安全的条件，分页使用 `Page<T>` 配合 `PaginationInnerInterceptor` 插件。

### 架构分层

```
Controller (OrderController)
    ↓ 构造器注入
Service 接口 (OrderService)
    ↓
ServiceImpl (OrderServiceImpl)
    ↓ 构造器注入
Mapper (ShopOrderMapper extends BaseMapper<ShopOrder>)
    ↓
Database (SQLite/MySQL)
```

### 关键设计决策

1. **构造器注入代替 @Autowired**：遵循 ECC 规范，提高可测试性
2. **LambdaQueryWrapper**：类型安全的查询条件，避免字符串拼写错误
3. **声明式事务**：`@Transactional(rollbackFor = Exception.class)` 保证下单扣库存的原子性
4. **无状态 JWT**：用 `HandlerInterceptor` 替代 Shiro，每次请求独立验证
5. **分页插件**：`PaginationInnerInterceptor` 自动处理分页 SQL

## 3. 订单状态流转

```
PENDING → SHIPPED → COMPLETED
    ↓
CANCELLED
```

只有管理员可以更改订单状态，普通用户只能查看自己的订单。
