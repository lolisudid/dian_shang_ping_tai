# 登录 Token 验证原理说明

## 一、整体架构

本项目采用 **JWT（JSON Web Token）+ Shiro 无状态认证** 方案，前后端分离。

## 二、登录流程

### 1. 用户提交登录表单
前端 Login.vue 收集用户名密码，通过 Axios 发送 POST 请求：
POST /api/auth/login  { username, password }

### 2. 后端验证密码
- AuthController.login() 接收请求，调用 AuthServiceImpl.login()
- AuthServiceImpl 通过 UserMapper 查询数据库，使用 **BCryptPasswordEncoder** 比对密码哈希
- 密码错误或用户不存在 → 抛出 BusinessException(401)

### 3. 生成 JWT Token
- 密码验证通过后，JwtUtils.createToken() 生成 JWT：
  - **载荷 (Payload)**：{ userId, username, role }
  - **签名算法**：HS256（HMAC-SHA256）
  - **过期时间**：由配置文件 jwt.expire-hours 控制，默认 24 小时
  - **密钥**：来自配置文件 jwt.secret
- 返回 { token, userId, username, role } 给前端

### 4. 前端存储 Token
- Login.vue 调用 useUserStore().setUser(res.data) 将 token/userId/username/role
  存入 **Pinia Store** 和 **localStorage**

## 三、Token 验证流程（每次请求）

### 请求发送（前端）
- equest.js 的 Axios **请求拦截器** 自动从 Pinia Store 读取 token
- 添加 HTTP Header：Authorization: Bearer <token>

### 请求拦截（后端）
- **Shiro Filter Chain**：ShiroConfig 配置路径权限
  - /api/auth/login、/api/auth/register → non（无需认证）
  - /api/products/**、/api/reviews/product/** → non（公开浏览）
  - /api/** → jwt（需要 JWT 验证）
- **JwtFilter**（继承 BasicHttpAuthenticationFilter）：
  1. 从 Header 提取 Bearer 后的 token 字符串
  2. 构造 JwtToken 对象
  3. 调用 getSubject(request, response).login(jwtToken) 触发认证

### Token 校验（后端）
- **JwtRealm**（继承 AuthorizingRealm）：
  1. doGetAuthenticationInfo()：解析 JWT，验证签名和过期时间
  2. 如果 token 无效/过期 → 抛出 AuthenticationException → JwtFilter 返回 401
  3. 如果验证通过 → 创建 UserPrincipal(userId, username, role) 存入 Shiro Subject
- **UserContext** 工具类：从 SecurityUtils.getSubject().getPrincipal() 获取 UserPrincipal
  - currentUserId() → 获取当前用户 ID（未登录抛 401）
  - isAdmin() → 判断是否管理员

### 认证失败处理（前端）
- equest.js 的 Axios **响应拦截器** 捕获 401 状态码
- 自动调用 user.logout() 清除登录状态
- 重定向到登录页，并携带 ?redirect=原路径 参数

## 四、核心类职责

| 类 | 职责 |
|---|---|
| JwtUtils | 生成/解析 JWT Token |
| JwtFilter | Shiro 过滤器，从请求头提取 Token 并触发登录 |
| JwtToken | Shiro Token 类型封装 |
| JwtRealm | JWT 认证与授权逻辑 |
| UserPrincipal | 存储 userId/username/role 的 Principal |
| UserContext | 从 Shiro 获取当前用户信息的静态工具类 |
| ShiroConfig | Shiro 配置，路径权限，无状态 Session |
| request.js | Axios 封装，自动携带 Token，统一处理 401 |
| user.js | Pinia Store，管理登录状态 |

## 五、安全保障

- 密码使用 BCrypt 哈希存储，不可逆
- JWT 使用 HS256 签名，防篡改
- Token 有过期时间，降低泄露风险
- Shiro 禁用 Session 存储，真正无状态
