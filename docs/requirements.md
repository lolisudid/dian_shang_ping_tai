

## 电商购物平台开发需求

### 1. 技术栈
#### （1）系统环境
- Java EE 8
- Servlet 3.0
- Apache Maven 3

#### （2）主框架
- Spring Boot 2.2.x
- Spring Framework 5.2.x
- Apache Shiro 1.7

#### （3）持久层
- Apache MyBatis 3.5.x
- Hibernate Validation 6.0.x
- Alibaba Druid 1.2.x

#### （4）视图层
- Bootstrap 3.3.7
- Vue 3
- （可补充：Axios 用于前后端异步交互）

---

### 2. 数据库支持
支持 **SQLite** 和 **MySQL** 两种数据库，并可在配置文件中自由切换。
**初期版本默认使用 SQLite**，以简化开发环境搭建，后续可平滑升级至 MySQL 生产环境。

---

### 3. 业务功能
**核心业务：商品信息管理（含 AI 辅助内容生成）**

- **商品信息录入**
  提供后台管理界面，管理员可录入商品基本信息，包括商品名称、分类、价格、库存、商品图片（URL）等。
  **商品描述**由 AI 自动生成（例如调用 OpenAI API 或本地模型，根据商品名称和分类生成一段商品介绍文案），生成后也可人工编辑。

- **商品信息查询**
  支持分页、多条件模糊查询（按名称、分类、价格区间），前端展示商品列表及详情。

- **商品信息修改**
  管理员可修改商品的各项属性，修改后重新校验。

- **商品信息删除**
  支持逻辑删除（下架）或物理删除，删除前进行关联检查（如该商品已有未完成订单时给出提示）。

- **补充业务（之前遗漏）**
  - **用户注册/登录**：采用 Shiro 进行身份认证与授权，支持普通用户和管理员两种角色。
  - **购物车功能**：用户可将商品加入购物车，修改数量、删除项。
  - **订单模块**：用户提交订单，扣减库存，生成订单记录；管理员可查看、修改订单状态（发货等）。
  - **商品评价**：用户对已购商品进行评价，支持评分和文字评论。

### 4. 服务层与数据操作层设计规范
- **项目分包结构遵循接口与实现分离：**
  - `com.xxx.service` 接口包：存放业务服务接口（如 `ProductService`, `OrderService`）。
  - `com.xxx.service.impl` 实现包：存放对应业务实现类。
  - `com.xxx.dao` 接口包：MyBatis Mapper 接口（数据访问层）。
  - `com.xxx.entity` 实体类包：与数据库表对应的 POJO。
- **严格遵守分层调用**：Controller → Service 接口 → ServiceImpl → Mapper。
- **统一返回结果封装**：设计 `Result` 类，包含 `code`, `message`, `data` 三个字段，所有控制器返回此格式。

---

### 5. 拦截器设计
- **建立独立拦截器包**：`com.xxx.interceptor`
- **必备拦截器及功能：**
  - **请求日志拦截器**：打印所有请求的 URL、方法、参数、IP 及处理耗时。
  - **登录认证拦截器（由 Shiro 实现）**：对非公开接口进行权限校验，未登录跳转登录页或返回 401。
  - **跨域处理拦截器**：添加 CORS 响应头，允许前端跨域访问（开发阶段必须）。
- **执行时打印信息示例：**
  ```
  [Request] GET /api/products?page=1&size=10 | IP: 192.168.1.100 | Params: page=1, size=10
  [Response] 200 OK | Time: 45ms
  ```

### 6. 补充说明（原需求遗漏但实际需要）
- **前端路由与状态管理**
  - 使用 Vue Router 管理页面跳转（首页、商品列表、详情、购物车、订单、登录注册等）。
  - 使用 Vuex 或 Pinia 集中管理用户 token、购物车信息等全局状态。
- **统一异常处理**

- **文件上传**（扩展）
  商品图片可通过后台上传到服务器，返回可访问 URL 存入数据库。
- **API 文档**
  建议集成 Swagger，自动生成后端接口文档，方便前后端联调。
