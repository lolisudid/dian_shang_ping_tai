# Cursor 项目开发指令（电商购物平台）

## 1. 技术栈严格遵循
- Spring Boot 2.2.x, Spring Framework 5.2.x, Apache Shiro 1.7
- MyBatis 3.5.x, Druid 1.2.x, Hibernate Validator 6.0.x
- 数据库：SQLite（开发环境），可配置切换 MySQL
- 前端：Vue 3 + Bootstrap 3.3.7，使用 Axios

## 2. 项目包结构规范
- 基础包：com.ecommerce
- 实体类：com.ecommerce.entity
- Mapper：com.ecommerce.mapper（接口），XML 放在 resources/mapper
- Service 接口：com.ecommerce.service
- Service 实现：com.ecommerce.service.impl
- Controller：com.ecommerce.controller
- 拦截器：com.ecommerce.interceptor
- 工具类：com.ecommerce.util
- 统一返回类：com.ecommerce.common.Result<T>

## 3. 编码规范
- 所有 Controller 返回值必须使用 Result<T> 封装（包含 code, message, data）
- 使用 @Valid 进行参数校验，异常由全局处理器捕获
- Service 接口与实现分离
- 实体类使用 Lombok 简化 getter/setter
- 所有 Mapper 必须用 @Mapper 或启动类添加 @MapperScan

## 4. 业务功能执行顺序
必须按以下顺序逐模块生成，每次只生成一个模块的完整代码：
① 项目初始化（pom.xml, application.yml, 启动类）
② 数据库表设计+实体类+Mapper（商品、用户、购物车、订单、评价）
③ 通用组件（Result、全局异常处理、请求日志拦截器）
④ 商品管理模块（Service+Controller，含 AI 生成描述的假实现）
⑤ 用户认证模块（Shiro + JWT，登录接口，权限拦截）
⑥ 购物车模块
⑦ 订单模块
⑧ 评价模块

## 5. 拦截器要求
- 请求日志拦截器：打印请求 URL、方法、IP、参数、耗时
- 所有拦截器统一放入 com.ecommerce.interceptor 包

## 6. 前端生成要求
- 使用 Vue Router 管理路由，路由守卫实现登录拦截
- 使用 Pinia 管理登录 token 和购物车状态
- Axios 实例拦截器自动携带 token，统一处理 401
- 组件使用 Bootstrap 3 样式类

## 7. 对话生成规则
- 当我询问时，你必须先检查我是否 @ 引用了需求文档，如未引用，主动提醒我
- 每一轮生成代码后，必须列出创建或修改的文件清单
- 生成 Controller 前，先确认 Service 接口已存在
- 所有 SQL 语句必须兼容 SQLite
- 如遇到不确定的业务细节，向我确认后继续
-生成完所有代码后给我生成一个框架文件，并解释每个文件的作用
-给所有代码添加通俗易懂的注释，并解释每个注释的作用
